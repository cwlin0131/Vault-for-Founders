# After Action Review

> Run this after every important task. If you did it but didn't record it, it's as good as not done.

---

## When to Trigger

- Completed an important decision
- Finished a project phase
- Wrapped up an important meeting or event
- Hit a snag, learned a lesson
- Anything you'll want to look back on later

---

## Three-Step Closeout

### Step 1: Write a Memory Record

Create a new file in `memory/`:

```
# [Brief Title]

Date: YYYY/MM/DD

## What Happened
[One or two sentences]

## Decision / Conclusion
[What was decided, and why]

## Lesson (if any)
[What went wrong, how to avoid it next time]
```

### Step 2: Update Related Files

Did this outcome affect other files in the Vault?

- Decision changed direction → update `context/`
- Learned a new pattern → update `identity/`
- Met someone important → update `people/`
- Project progressed → update `projects/`

### Step 3: Update Memory Summary

If it's significant enough, add it to the relevant table in `memory-summary.md`.

---

## Agent's Role

The Agent proactively reminds you to run the after-action review, especially when:
- You just made an important decision but didn't write it down
- A project wrapped up but the Vault wasn't updated
- You mentioned a lesson but didn't record it

Suggested prompt: "Want to record this? I can write it into memory for you."
